## Accessibility Notes

**WCAG contrast requirements:**

| Level | Normal text (< 24px) | Large text (>= 24px or >= 18.66px bold) |
|-------|----------------------|------------------------------------------|
| AA | 4.5:1 minimum | 3:1 minimum |
| AAA | 7:1 minimum | 4.5:1 minimum |

**Safe text combinations for light mode:**
- Shade 700+ on white (#ffffff): passes AA for normal text
- Shade 600+ on shade-50 backgrounds: passes AA for normal text
- Shade 900 on white: passes AAA for normal text

**Safe text combinations for dark mode:**
- Shade 50-200 on shade-900+: passes AA for normal text
- White (#ffffff) on shade-800+: passes AAA for normal text

**Non-text contrast (icons, borders, controls):** minimum 3:1 against adjacent colors.

## Dark Mode Generation Rules

When generating dark mode variants from a light palette:

1. **Invert the neutral scale** -- light bg becomes dark, dark text becomes light
2. **Keep primary hue** -- shift to 400-500 range (lighter shade) for visibility on dark
3. **Reduce saturation slightly** -- overly vivid colors cause eye strain on dark backgrounds
4. **Background layers:** Base (darkest, 900-950) > Surface (+1 step lighter) > Elevated (+2 steps)
5. **Borders:** Use `rgba(255, 255, 255, 0.08-0.15)` instead of solid colors
6. **Semantic colors:** Shift to lighter shades (400-500) for visibility; keep meaning unchanged
7. **Shadows:** Use `rgba(0, 0, 0, 0.3-0.5)` -- shadows must be darker than the surface

```css
/* Dark mode token override pattern */
@media (prefers-color-scheme: dark) {
  :root {
    --bg-base: #09090b;
    --bg-surface: #18181b;
    --bg-elevated: #27272a;
    --border-color: rgba(255, 255, 255, 0.1);
    --text-primary: #fafafa;
    --text-secondary: #a1a1aa;
    --color-primary: #818cf8; /* shifted from 600 to 400 */
  }
}
```
